## Overview

Inhibitors are middleware that run before a command executes. They can allow or block a command from running based on any condition you define. Permissions, cooldowns, disabled state, channel restrictions, and more.

## Execution flow

When a user sends a command, Flint runs all registered inhibitors in order before executing the command:
```
Message received
    -> Monitors run (on every message)
    -> Command parsed (prefix + name matched)
    -> Inhibitors run (can block execution)
    -> Command executes
```

If any inhibitor returns a failure result, the command is blocked and a `CommandDenied` event is emitted. No further inhibitors are run after a failure.

## Registering inhibitors

Inhibitors are registered on the `InhibitorHandler`. You need to add it to your client and link it to your `CommandHandler`:
```ts src/index.ts
import {
    FlintClient,
    CommandHandler,
    ListenerHandler,
    InhibitorHandler,
    ChannelTypeInhibitor,
    UserPermissionsInhibitor,
    BotPermissionsInhibitor
} from "@flint.js/core"

class MyClient extends FlintClient {
    commandHandler: CommandHandler
    listenerHandler: ListenerHandler
    inhibitorHandler: InhibitorHandler

    constructor() {
        super()

        this.inhibitorHandler = new InhibitorHandler(this, {
            directory: "./src/inhibitors",
            builtins: [
                new ChannelTypeInhibitor(),
                new UserPermissionsInhibitor(),
                new BotPermissionsInhibitor()
            ]
        })

        this.listenerHandler = new ListenerHandler(this, {
            directory: "./src/listeners"
        })

        this.commandHandler = new CommandHandler(this, {
            directory: "./src/commands",
            prefix: "!"
        })

        this.commandHandler
            .useListenerHandler(this.listenerHandler)
            .useInhibitorHandler(this.inhibitorHandler)
    }

    override async login(token: string) {
        await this.inhibitorHandler.loadAll()
        await this.listenerHandler.loadAll()
        await this.commandHandler.loadAll()
        return super.login(token)
    }
}
```

# Internal inhibitors

Some inhibitors are registered by default internally to ensure command execution works as intended with the provided command options. These are:

### OwnerOnly

Blocks a command from running if its `ownerOnly` property is set to `true` and `client.owners` doesn't include the id of the user executing the command.
```ts
export default defineCommand({
    name: "ping",
    description: "Pong!",
    ownerOnly: true,

    async execute(client, message) {
        // only runs if the message author's ID is included in `client.owners`
    }
})
```


When blocked, the `result` parameter within the `CommandDenied` event will be:

```ts
{
    reason: "ownerOnly"
}
```

### Disabled

Blocks a command from running if its `disabled` property is set to `true`.
```ts
export default defineCommand({
    name: "ping",
    description: "Pong!",
    disabled: true,

    async execute(client, message) {
        // only runs if the command is not `disabled`
    }
})
```

When blocked, the `result` parameter within the `CommandDenied` event will be:

```ts
{
    reason: "disabled"
}
```

### Cooldown

Blocks a command from running if it has a `cooldown` property set and the cooldown is still active when a user runs the command again.
```ts
export default defineCommand({
    name: "ping",
    description: "Pong!",
    cooldown: 5000,

    async execute(client, message) {
        // only runs if the command is not on cooldown for the user who executed the command
    }
})
```

When blocked, the `result` parameter within the `CommandDenied` event will be:

```ts
{
    reason: "cooldown",
    remaining: 2040,
    formatted: "2 seconds"
}
```

## Handling denied commands

When an inhibitor blocks a command, Flint emits a `CommandDenied` event with the result and context. Listen to this event to send feedback to the user:
```ts src/listeners/commandDenied.ts
import { defineListener, FlintClientEvents } from "@flint.js/core"

export default defineListener({
    name: "commandDenied",
    event: FlintClientEvents.CommandDenied,

    async execute(client, { result, ctx }) {
        switch (result.reason) {
            case "disabled":
                return ctx.message.reply("That command is currently disabled.")
            case "ownerOnly":
                return ctx.message.reply("This command is owner only.")
            case "channel":
                return ctx.message.reply("That command cannot be used in this channel.")
            case "user.permissions":
                return ctx.message.reply(`You are missing the following permissions: ${result.missing?.join(", ")}`)
            case "bot.permissions":
                return ctx.message.reply(`I am missing the following permissions: ${result.missing?.join(", ")}`)
            case "cooldown":
                return ctx.message.reply(`Please wait ${result.formatted} before using that command again.`)
        }
    }
})
```
