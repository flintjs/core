## Overview

Handlers are responsible for loading and managing pieces of your bot. Commands, listeners, inhibitors, and monitors. Each handler scans a directory, imports the files it finds, and stores the pieces ready for use.

All handlers extend `BaseHandler` and share the same core behaviour. Only their specific functionality differs.

## Available handlers

| Handler | Manages | Required |
|---|---|---|
| `CommandHandler` | Commands | Yes |
| `ListenerHandler` | Listeners | Yes |
| `InhibitorHandler` | Inhibitors | No |
| `MonitorHandler` | Monitors | No |

## Creating a handler

All handlers follow the same pattern, instantiate with a client reference and an options object:
```ts
this.commandHandler = new CommandHandler(this, {
    directory: "./src/commands",
    prefix: "!",
    mentionPrefix: true
})

this.listenerHandler = new ListenerHandler(this, {
    directory: "./src/listeners"
})

this.inhibitorHandler = new InhibitorHandler(this, {
    directory: "./src/inhibitors",
    builtins: [
        new Disabled(),
        new ChannelType(),
        new UserPermissions(),
        new BotPermissions()
    ]
})

this.monitorHandler = new MonitorHandler(this, {
    directory: "./src/monitors"
})
```

## Handler options

All handlers accept these base options:

| Option | Type | Default | Description |
|---|---|---|---|
| `directory` | `string` | - | Path to the directory to load files from |
| `recursive` | `boolean` | `true` | Whether to scan subdirectories |

### CommandHandler options

| Option | Type | Default | Description |
|---|---|---|---|
| `prefix` | `string` | - | The command prefix |
| `mentionPrefix` | `boolean` | `false` | Whether to allow mentioning the bot as a prefix |

### InhibitorHandler options

| Option | Type | Default | Description |
|---|---|---|---|
| `builtins` | `BaseInhibitor[]` | `[]` | Built-in inhibitors to register on startup |

## Linking handlers

Some handlers depend on others. Use the `use` methods to link them before calling `loadAll()`:
```ts
this.commandHandler
    .useListenerHandler(this.listenerHandler)
    .useInhibitorHandler(this.inhibitorHandler)
    .useMonitorHandler(this.monitorHandler)
```

These methods return `this` so they can be chained. The `CommandHandler` uses the linked handlers internally when processing commands.

## Loading

Call `loadAll()` on each handler inside your client's `login()` override. Load order matters. Handlers that others depend on should be loaded first:
```ts
override async login(token: string) {
    await this.inhibitorHandler.loadAll()
    await this.listenerHandler.loadAll()
    await this.commandHandler.loadAll()
    return super.login(token)
}
```

## Accessing pieces

Each handler exposes `get` and `getAll` methods for accessing loaded pieces:
```ts
// get a specific command by name
const command = client.commandHandler.get("ping")

// get all loaded commands
const commands = client.commandHandler.getAll()
```

## Adding a logger

Attach a logger to any handler to get styled output when pieces are loaded:
```ts
this.commandHandler.useLogger(this.logger)
this.listenerHandler.useLogger(this.logger)
```
