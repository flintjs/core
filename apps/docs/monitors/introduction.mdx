## Overview

Monitors run on every message your bot receives, before any command parsing takes place. They are useful for things like spam filtering, message logging, invite detection, or any logic that needs to watch all incoming messages rather than just commands.

## Execution flow
```
Message received
    -> Monitors run (on every message, before command parsing)
    -> Command parsed (prefix + name matched)
    -> Inhibitors run (can block execution)
    -> Command executes
```

Unlike inhibitors, monitors cannot block command execution. They run independently and in parallel with the rest of the message handling flow.

## Registering monitors

Monitors are registered on the `MonitorHandler`. Add it to your client and link it to your `CommandHandler`:
```ts src/index.ts
import {
    FlintClient,
    CommandHandler,
    ListenerHandler,
    MonitorHandler
} from "@flint.js/core"

class MyClient extends FlintClient {
    commandHandler: CommandHandler
    listenerHandler: ListenerHandler
    monitorHandler: MonitorHandler

    constructor() {
        super()

        this.monitorHandler = new MonitorHandler(this, {
            directory: "./src/monitors"
        })

        this.listenerHandler = new ListenerHandler(this, {
            directory: "./src/listeners"
        })

        this.commandHandler = new CommandHandler(this, {
            directory: "./src/commands",
            prefix: "!"
        })

        this.commandHandler
            .useListenerHandler(this.listenerHandler)
            .useMonitorHandler(this.monitorHandler)
    }

    override async login(token: string) {
        await this.monitorHandler.loadAll()
        await this.listenerHandler.loadAll()
        await this.commandHandler.loadAll()
        return super.login(token)
    }
}
```

## Monitor options

All monitors support the following options which control when they run:

| Option | Type | Default | Description |
|---|---|---|---|
| `allowBots` | `boolean` | `false` | Whether to run on messages from bots |
| `allowDMs` | `boolean` | `false` | Whether to run on direct messages |
| `disabled` | `boolean` | `false` | Whether the monitor is disabled |
