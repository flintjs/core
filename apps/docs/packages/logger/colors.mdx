## Overview

`@flint.js/logger` exports a `colors` utility that lets you add color and styling to strings. This is useful for highlighting important information in your log messages.
```ts
import { colors } from "@flint.js/logger"

client.logger.info(`Command ${colors.bold("ping")} used by ${colors.cyan(message.author.username)}`)
client.logger.warn(`Missing permission: ${colors.red("BanMembers")}`)
client.logger.success(`Loaded ${colors.bold(count.toString())} commands`)
```

## Available colors

| Function | Description |
|---|---|
| `colors.bold(text)` | Bold text |
| `colors.dim(text)` | Dimmed text |
| `colors.red(text)` | Red text |
| `colors.green(text)` | Green text |
| `colors.yellow(text)` | Yellow text |
| `colors.blue(text)` | Blue text |
| `colors.cyan(text)` | Cyan text |
| `colors.magenta(text)` | Magenta text |
| `colors.gray(text)` | Gray text |
| `colors.white(text)` | White text |

## Usage in commands and listeners

Colors work anywhere you have access to a string, combine them freely:
```ts
import { defineListener, FlintClientEvents } from "@flint.js/core"
import { colors } from "@flint.js/logger"

export default defineListener({
    name: "commandSuccess",
    event: FlintClientEvents.CommandSuccess,

    async execute(client, { ctx }) {
        client.logger.info(
            `${colors.bold(ctx.meta.command.name)} used by ${colors.cyan(ctx.message.author.username)} in ${colors.gray(ctx.message.guild?.name ?? "DMs")}`
        )
    }
})
```
```ts
import { defineListener, FlintClientEvents } from "@flint.js/core"
import { colors } from "@flint.js/logger"

export default defineListener({
    name: "commandDenied",
    event: FlintClientEvents.CommandDenied,

    async execute(client, { result, ctx }) {
        client.logger.warn(
            `${colors.bold(ctx.meta.command.name)} denied for ${colors.cyan(ctx.message.author.username)}: ${colors.red(result.reason)}`
        )
    }
})
```

## Combining colors

Colors can be combined by nesting them:
```ts
// bold and red
colors.bold(colors.red("Error!"))

// bold and cyan
colors.bold(colors.cyan("Success!"))
```

## Without the logger

You can use `colors` independently of the `Logger` class. It just returns a formatted string, so it works anywhere:
```ts
import { colors } from "@flint.js/logger"

console.log(`Hello, ${colors.cyan("world")}!`)

await message.reply(`Welcome, ${colors.bold(member.user.username)}!`)
```

<Note>
    Color codes are ANSI escape sequences and will only render correctly in terminals that support them. They will appear as raw escape characters in environments that do not support ANSI, such as some log file viewers.
</Note>
