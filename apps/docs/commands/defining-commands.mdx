## Overview

Commands are the most fundamental piece of a Flint bot. When a user sends a message with your bot's prefix followed by a command name, Flint finds the matching command and runs its `execute` function.

## Using defineCommand

The easiest way to define a command is with the `defineCommand` factory:
```ts src/commands/ping.ts
import { defineCommand } from "@flint.js/core"

export default defineCommand({
    name: "ping",
    description: "Pong!",

    async execute(client, message, args, ctx) {
        await message.reply("Pong! 🏓")
    }
})
```

Each command file should have a default export of a `defineCommand` call.

## Command options

| Option | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | Yes | The name of the command |
| `description` | `string` | Yes | A short description of what the command does |
| `category` | `string` | No | The category the command belongs to, defaults to `Uncategorized` |
| `aliases` | `string[]` | No | Alternative names the command can be called by |
| `cooldown` | `number` | No | Cooldown in milliseconds between uses |
| `permissions` | `PermissionResolvable[]` | No | Permissions required to run the command |
| `allowedChannels` | `string[]` | No | Channel IDs the command can be run in |
| `prefixes` | `string[]` | No | Per-command prefixes, overrides the global prefix |
| `disabled` | `boolean` | No | Whether the command is disabled |
| `args` | `ArgumentOptions[]` | No | Argument definitions for the command |

## The execute function

The `execute` function receives four parameters:

| Parameter | Type | Description |
|---|---|---|
| `client` | `FlintClient` | Your bot client |
| `message` | `Message` | The message that triggered the command |
| `args` | `Record<string, unknown>` | Parsed arguments, typed if `args` is defined |
| `ctx` | `CommandMeta` | Metadata about the invocation |

`CommandMeta` contains the following:

| Property | Type | Description |
|---|---|---|
| `prefix` | `string` | The prefix used to trigger the command |
| `usedAlias` | `string \| null` | The alias used, if any |
| `command` | `BaseCommand` | The command that was executed |

## Typing the client

By default, `client` in `execute` is typed as `FlintClient`. If your client extends `FlintClient` with custom properties or handlers, annotate `client` directly in `execute`:
```ts
import { defineCommand } from "@flint.js/core"
import type { MyClient } from "../index"

export default defineCommand({
    async execute(client: MyClient, message, args, ctx) {
        // client is now typed as MyClient
        // message, args, and ctx are still inferred automatically
        client.i18n.t("PING_RESPONSE")
    }
})
```

Only `client` needs to be annotated, all other parameters are inferred automatically.

<Tip>
    If most of your commands need access to custom client properties, consider creating a barrel export for your client type so you don't have to navigate to a specific path each time.
</Tip>

## Aliases

Commands can be triggered by any of their aliases:
```ts
export default defineCommand({
    name: "help",
    description: "Show help",
    aliases: ["h", "?"],

    async execute(client, message) {
        await message.reply("Here is the help menu!")
    }
})
```

## Per-command prefixes

You can override the global prefix for specific commands:
```ts
export default defineCommand({
    name: "config",
    description: "Configure the bot",
    prefixes: [">>"],

    async execute(client, message) {
        await message.reply("Config command!")
    }
})
```

## Adding extra methods

Since `defineCommand` is generic, you can add your own helper methods directly on the command object and access them via `this`:
```ts
export default defineCommand({
    name: "help",
    description: "Show help",

    formatList(items: string[]) {
        return items.map((item, i) => `${i + 1}. ${item}`).join("\n")
    },

    async execute(client, message) {
        const commands = client.commandHandler.getAll()
        const list = this.formatList(commands.map(c => c.name))
        await message.reply(list)
    }
})
```
